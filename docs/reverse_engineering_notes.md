# ROB3 Firmware Reverse Engineering Notes

## Firmware Overview

- **Source:** `firmware/bin/M2764A@DIP28.BIN` (8192 bytes = 8 KB EPROM)
- **Disassembly:** `firmware/src/main.asm` (7183 lines, generated by disasm51.py)
- **Architecture:** Intel 8031/MCS-51
- **Code space:** 0x0000 - 0x0FDE (actual code), remainder is 0xFF padding (disassembled as `mov R7, A`)
- **External RAM:** Detected, size auto-probed at startup

### External Hardware (MOVX addresses)

The 8031 uses MOVX with DPTR where DPH (0x83) selects the "device" and DPL (0x82) provides data/address within the device.

| DPH Value | Device/Function |
|-----------|-----------------|
| 0x48      | Axis select register (write: select active axis 0-7) |
| 0x50      | 8255 Port A (stepper motor output - even axes?) |
| 0x51      | Digital output port (general purpose I/O) |
| 0x52      | 8255 Port C (stepper motor output - odd axes?) |
| 0x53      | 8255 Control register |
| 0x58      | Axis feedback read (encoder/position input) |
| 0x59      | Axis position read #2 |
| 0xA0+     | External RAM (program storage) - upper bound probed |

## Interrupt System

### Reset Vector (0x0000)
```
ljmp jump_05FF   ; Jump to initialization
```

### External Interrupt 0 (0x0003) → jump_003F
- Stepper motor pulse output ISR
- Writes motor control to 8255 Port A (DPH=0x50) and Port C (DPH=0x52)
- Uses register bank 1 (PSW.3 = 1)
- Reads from RAM locations 0x4E, 0x4F (motor output shadows)
- Handles debounce on P3.2 input
- Sets flag 20h.1 (keyboard event) or sets 47h=0xFF (emergency?)

### Timer 0 (0x000B) → jump_0080
- Periodic tick interrupt
- Reloads timer: TH0=0x11, TL0=0xE8 (period ≈ 5ms @ 11.0592 MHz)
- Toggles flag 20h.3 (used for timing), sets 20h.4 (axis update request)
- Prescaler at 0x1D counts down from 10 → fires 23h.6, 23h.5 every 50ms

### External Interrupt 1 (0x0013) → jump_00BF
- **Axis control state machine** (complex ISR)
- Uses register bank 1 (PSW.3 = 1)
- Reads axis feedback from DPH=0x59
- Performs PID-like servo calculations
- Updates motor outputs to DPH=0x50 / DPH=0x52
- Manages acceleration/deceleration profiles
- Rotates through all 6 axes (round-robin via 22h mask)
- Writes axis select to DPH=0x58

### Timer 1 (0x001B) → jump_02FF
- **Serial communication handler** (baud rate generator in mode 2)
- Uses register bank 2 (PSW.4 = 1)
- Handles both RX (98h.0 = RI) and TX (98h.1 = TI)
- Implements a multi-state protocol parser
- Commands begin with 0x89-0x8F range bytes
- 0x81 = special marker (start of data block)
- Protocol supports: axis position commands, program upload/download, I/O control

## System Initialization (jump_05FF)

```
1. Delay loop (warm-up)
2. Write 0x00 to DPH=0x48 (axis select = 0)
3. Configure 8255: Write 0x80 to DPH=0x53 (Mode 0, all ports output)
4. Set Port A (DPH=0x50) = 0x00
5. Set Port B (DPH=0x51) = 0xFF
6. Set Port C (DPH=0x52) = 0x00
7. Write 0x01 to DPH=0x59 (axis feedback init?)
8. Clear all internal RAM (0x00-0x7F)
9. Set SP = 0x31
10. Set 47h = 0xFF, 1Fh = 0xFF
11. Probe external RAM size:
    - Start at DPH=0x7F (DPTR=7FFF)
    - Write complement, read back, verify
    - If fails, try DPH=0xA0
    - Store RAM page in 3Eh, set 3Fh = 3Eh+1
12. Initialize program header in external RAM
13. Baud rate auto-detection:
    - If P3.0 = 0: fast mode (IE=0x07, set 20h.2, jump to main loop)
    - Otherwise: measure incoming signal timing on P3.0
    - Calculate baud rate divisor → TH1
    - Enable Timer 1 for baud generation
14. Enable interrupts (IE=0x17 or 0x07)
15. Start Timer 0
16. Enter main loop
```

## Main Loop (jump_074D)

```
LOOP:
    CLI (disable interrupts)
    Select register bank 2

    // Check axis motion completion
    if (flag 25h.7):
        if all_axes_at_target:
            signal completion
        elif timeout:
            emergency stop (disable motors, clear outputs)

    // Service serial TX if ready
    if (flag 23h.0):
        call serial_tx_handler (jump_0541)

    // Check serial timeout
    if (timer tick 23h.7):
        if serial_active && timeout:
            reset serial state

    // Deselect register bank 2
    call motion_executor (jump_08FF)

    STI (enable interrupts)

    // Periodic axis update (on timer flag 20h.4)
    if (flag 20h.4 && flag 20h.2):
        clear 20h.4
        if NOT motion_active:
            if P3.4 low:
                init_program_from_ram
            else:
                call teach_pendant_scan (jump_0BFF)
                if (key detected):
                    call teach_pendant_handler (jump_0C7F)

    goto LOOP
```

## Communication Protocol

### Serial Configuration
- Mode: SCON = 0x50 (Mode 1, 8-bit UART, REN enabled)
- Baud rate: Auto-detected from incoming signal
- Timer 1 in Mode 2 (8-bit auto-reload) for baud generation

### Protocol Format
Commands from host begin with a header byte in range 0x80-0xFF:
- Bit 7 = 1: Command byte
- Bit 6: Command class (0=axis/position, 1=system/program)
- Bits 5-4: Sub-command type
- Bits 2-0: Axis number (0-5) or 0x07 for all axes

### Identified Commands

| Byte Pattern | Meaning |
|-------------|---------|
| 0x87 (0_000_0_111) | Request all axis positions |
| 0x67 (0_110_0_111) | Set all axis positions (followed by 6 data bytes) |
| 0x6N (0_110_0_NNN) | Set single axis N position |
| 0x77 (0_111_0_111) | Set all axis targets + speed |
| 0x7N (0_111_0_NNN) | Set single axis N target |
| 0x81 | Data block start marker |
| 0x89-0x8F | Program download initiation |
| 0x4N | Read axis N feedback |

### Response Format
- Responses sent via serial TX handler (jump_0541)
- Multi-byte responses framed with 0x03 (ETX) terminator
- Position data sent as raw bytes (1 byte per axis)

## 8255 PPI Configuration

### Base Addressing
The 8255 is memory-mapped via the 8031 external bus. DPH selects the register:

| DPH | 8255 Register |
|-----|---------------|
| 0x50 | Port A (output) |
| 0x51 | Port B (output) - used as general digital output |
| 0x52 | Port C (output) |
| 0x53 | Control Register |

### Initialization
```
Control = 0x80  → Mode 0, Port A output, Port B output, Port C output
Port A  = 0x00  → All motor outputs off
Port B  = 0xFF  → All digital outputs high (active-low?)
Port C  = 0x00  → All motor outputs off
```

### Port A Function
- Stepper motor control output (phase signals)
- Updated by External Interrupt 0 ISR
- Shadow register at RAM 0x4E

### Port B Function
- General digital output port
- Shadow register at RAM 0x1F
- Written via jump_07D0 helper function
- Supports: SET (OR), CLEAR (AND), TOGGLE (XOR), WRITE operations

### Port C Function
- Stepper motor control output (second set)
- Updated by External Interrupt 0 ISR
- Shadow register at RAM 0x4F

## Axis Control Architecture

### Axis Mechanical Limits (from documentation)

| Axis | Joint | Symbol | Range | Resolution |
|------|-------|--------|-------|------------|
| 0 | Base rotation | q1 | +80° to −80° | 0–255 (8-bit) |
| 1 | Shoulder | q2 | +70° to −30° | 0–255 (8-bit) |
| 2 | Elbow | q3 | 0° to −100° | 0–255 (8-bit) |
| 3 | Wrist | q4 | +100° to −100° | 0–255 (8-bit) |
| 4 | Wrist roll | q5 | +100° to −100° | 0–255 (8-bit) |
| 5 | Gripper | — | 0 to 60 mm | 0–255 (8-bit) |

Note: Firmware uses zero-based numbering. Original robot documentation uses one-based.

### Axis State Machine (External Interrupt 1 ISR)
The axis controller runs in the External Interrupt 1 handler, cycling through all 6 axes in a round-robin fashion.

For each axis:
1. Read current position from feedback (DPH=0x59)
2. Compare with target position
3. Calculate error
4. Apply acceleration/deceleration profile
5. Output motor command
6. Update axis flags (moving/stopped/at target)

### Axis Data Structure (per axis, indexed N=0..5)

| Address | Content |
|---------|---------|
| 0x40+N | Target position |
| 0x48+N | Speed/step rate |
| 0x50+N | Current position (host view) |
| 0x58+N | Encoder feedback value |
| 0x70+N | Deceleration profile |
| 0x78+N | ISR workspace |

### Axis Select Mechanism
- Write axis number (0-7) to DPH=0x58
- Axis selection stored in 0x48+N pattern (ORed with 0x48)
- The ISR rotates through axes via mask register 0x22

## Teach Pendant / Keyboard Interface

### Scanner (jump_0BFF)
- Reads from DPH=0x51 port
- Scans key matrix using output 0x47
- Detects key press with debounce (3-sample confirmation)
- Returns key code or 0 (no key)

### Command Handler (jump_0C7F)
Interprets key codes and implements:
- Axis jog (manual movement)
- Program editing (insert/delete/modify instructions)
- Program execution control (run/stop)
- Position teaching (store current position)
- Display control (LED/7-segment update via 0x47)

## Program Storage (External RAM)

### Structure
- Programs stored in external RAM starting at page 0x3E
- Header at offset 0xFE: program size (2 bytes)
- Instructions stored sequentially, 8 bytes each
- End marker: 0x83

### Instruction Format (8 bytes)
- Byte 0: Instruction type/opcode
- Bytes 1-6: Axis target positions (one per axis)
- Byte 7: Speed/flags

### Program Execution (jump_0941)
- Reads instruction from external RAM at address (0x66:0x67)
- Decodes instruction type from byte 0
- Types include: move, wait, I/O, jump, subroutine

## Unknown / Uncertain Areas

1. **Address 0x0023 (serial vector):** The disassembler shows `lcall jump_22FE` at this location, but 0x22FE is outside the 8KB ROM. This is likely a disassembly artifact where the interrupt vector overlaps with padding bytes.

2. **DPH=0x48 purpose:** Written during init with loop counter. May be an axis multiplexer/selector, or could be a secondary 8255 or latch.

3. **DPH=0x59 function:** Read during axis ISR. Likely encoder/position feedback, but exact hardware (counter, ADC, etc.) is unknown.

4. **P3.4 input:** Checked in main loop. May be a "program loaded" or "external trigger" signal.

5. **P3.0 input:** Used during baud rate detection. When low at startup, uses fixed fast baud rate. Otherwise, auto-detects.

6. **P3.2 input:** Checked in ext int 0 ISR for debounce. Possibly a step signal or home switch.

7. **Exact servo algorithm:** The math in isr_ext1 involves multiplication (MUL AB), table lookups (MOVC), and conditional branching that suggests a proportional control loop, but the exact gains and profile are not yet fully decoded.
